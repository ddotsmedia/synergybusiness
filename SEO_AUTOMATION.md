# SEO Automation — Synergy Business

Tailored SEO setup for the Synergy Business Next.js 14 (App Router) project.
Goal: every new page gets correct sitemap entries, metadata, JSON-LD, robots
rules, and OG image **automatically** — no manual steps per page.

Stack already in `apps/web`: Next.js 14 App Router, Tailwind, `next-sitemap`
(already installed), Sanity CMS planned.

---

## 1. What "automatic" means here

| Concern              | Mechanism                                                  |
| -------------------- | ---------------------------------------------------------- |
| sitemap.xml          | `next-sitemap` runs on `postbuild` — picks up every route. |
| robots.txt           | Generated by `next-sitemap` from a single config file.     |
| `<title>` / meta     | Per-route `generateMetadata()` + a shared helper.          |
| OpenGraph image      | `app/opengraph-image.tsx` (dynamic, per-route).            |
| Structured data      | JSON-LD helpers (`LocalBusiness`, `Service`, `FAQPage`).   |
| Canonical URLs       | Resolved in the metadata helper from `NEXT_PUBLIC_APP_URL`.|
| hreflang (en-AE)     | Set in root layout + sitemap config.                       |
| Search Console ping  | `next-sitemap` submission step on deploy.                  |

## 2. One-time setup

### 2.1 `next-sitemap.config.js` (repo root of `apps/web`)

```js
/** @type {import('next-sitemap').IConfig} */
module.exports = {
  siteUrl: process.env.NEXT_PUBLIC_APP_URL || 'https://synergybusiness.com',
  generateRobotsTxt: true,
  sitemapSize: 5000,
  changefreq: 'weekly',
  priority: 0.7,
  exclude: ['/portal/*', '/api/*', '/sign-in', '/sign-up'],
  robotsTxtOptions: {
    policies: [
      { userAgent: '*', allow: '/' },
      { userAgent: '*', disallow: ['/portal', '/api'] },
    ],
    additionalSitemaps: [
      `${process.env.NEXT_PUBLIC_APP_URL}/sitemap.xml`,
    ],
  },
  transform: async (config, path) => ({
    loc: path,
    changefreq: path === '/' ? 'daily' : 'weekly',
    priority: path === '/' ? 1.0 : path.startsWith('/services') ? 0.9 : 0.7,
    lastmod: new Date().toISOString(),
  }),
};
```

Add to `apps/web/package.json`:

```json
"scripts": {
  "postbuild": "next-sitemap --config next-sitemap.config.js"
}
```

### 2.2 Shared metadata helper — `apps/web/lib/seo.ts`

```ts
import type { Metadata } from 'next';

const SITE = {
  name: 'Synergy Business',
  url: process.env.NEXT_PUBLIC_APP_URL ?? 'https://synergybusiness.com',
  twitter: '@synergybusinessuae',
  locale: 'en_AE',
};

export function buildMetadata(opts: {
  title: string;
  description: string;
  path: string;          // e.g. '/services/free-zone'
  image?: string;        // override OG image
  noindex?: boolean;
}): Metadata {
  const url = `${SITE.url}${opts.path}`;
  return {
    title: opts.title,
    description: opts.description,
    alternates: { canonical: url },
    robots: opts.noindex ? { index: false, follow: false } : undefined,
    openGraph: {
      title: opts.title,
      description: opts.description,
      url,
      siteName: SITE.name,
      locale: SITE.locale,
      type: 'website',
      images: opts.image ? [{ url: opts.image }] : undefined,
    },
    twitter: {
      card: 'summary_large_image',
      title: opts.title,
      description: opts.description,
      images: opts.image ? [opts.image] : undefined,
    },
  };
}
```

Usage in any page:

```ts
// app/(marketing)/services/free-zone/page.tsx
import { buildMetadata } from '@/lib/seo';

export const metadata = buildMetadata({
  title: 'Free Zone Company Setup in Abu Dhabi',
  description: 'ADGM, KIZAD, twofour54, Masdar City — full setup from AED 12,500.',
  path: '/services/free-zone',
});
```

### 2.3 JSON-LD helpers — `apps/web/lib/jsonld.ts`

```ts
const SITE_URL = process.env.NEXT_PUBLIC_APP_URL ?? 'https://synergybusiness.com';

export const localBusinessLd = {
  '@context': 'https://schema.org',
  '@type': 'LocalBusiness',
  name: 'Synergy Business',
  url: SITE_URL,
  telephone: '+971-XX-XXX-XXXX',
  address: {
    '@type': 'PostalAddress',
    streetAddress: 'TBD',
    addressLocality: 'Abu Dhabi',
    addressCountry: 'AE',
  },
  areaServed: ['AE'],
  priceRange: 'AED',
};

export const serviceLd = (s: { name: string; description: string; slug: string }) => ({
  '@context': 'https://schema.org',
  '@type': 'Service',
  name: s.name,
  description: s.description,
  provider: { '@type': 'LocalBusiness', name: 'Synergy Business' },
  areaServed: { '@type': 'Country', name: 'United Arab Emirates' },
  url: `${SITE_URL}/services/${s.slug}`,
});

export const faqLd = (qa: { q: string; a: string }[]) => ({
  '@context': 'https://schema.org',
  '@type': 'FAQPage',
  mainEntity: qa.map(({ q, a }) => ({
    '@type': 'Question',
    name: q,
    acceptedAnswer: { '@type': 'Answer', text: a },
  })),
});
```

Drop into a page:

```tsx
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{ __html: JSON.stringify(localBusinessLd) }}
/>
```

### 2.5 Root layout additions

In `apps/web/app/layout.tsx` add to the existing `metadata` export:

```ts
metadataBase: new URL(process.env.NEXT_PUBLIC_APP_URL ?? 'https://synergybusiness.com'),
alternates: {
  canonical: '/',
  languages: { 'en-AE': '/' },
},
```

### 2.6 Google Search Console + Analytics

- Verify domain via `GOOGLE_SITE_VERIFICATION` meta tag in root metadata.
- Submit `https://synergybusiness.com/sitemap.xml` once after first deploy.
- Add GA4 via `@next/third-parties/google` — no manual script tags.

## 3. Per-page checklist (kept short on purpose)

When adding a new page:

1. Export `metadata = buildMetadata({...})` — done.
2. If the page lists FAQs, add `faqLd(...)` JSON-LD.
3. If the page is a service, add `serviceLd(...)` JSON-LD.
4. That's it — sitemap, robots, OG image, canonical are automatic.

## 4. Deploy hooks

`pnpm build` runs `postbuild` → regenerates `public/sitemap.xml` and
`public/robots.txt`. Vercel/VPS deploy picks them up. No extra step.

Optional: a GitHub Action that pings Google after each deploy:

## 6. What this does NOT do

- Doesn't write content / keywords for you.
- Doesn't auto-fix Core Web Vitals — that's a perf pass, separate.
- Doesn't do off-page SEO (backlinks, citations, GBP).
- Doesn't audit live pages — see "Recurring audit" below if wanted.
